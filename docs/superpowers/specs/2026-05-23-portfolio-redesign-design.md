# Portfolio Redesign — Design Spec

**Date:** 2026-05-23
**Site:** https://bobby-zs-lo.github.io/
**Repo:** Bobby-zs-lo.github.io (static GitHub Pages, no build step)
**Owner:** Bobby Zhao Sheng Lo, MD PhD

## 1. Goal

Replace the current academic site with a premium, dynamic, animated portfolio that signals "modern clinician-scientist" to clinicians, academic collaborators, and grant reviewers — without breaking the GitHub Actions auto-update flow that regenerates `publications.html` weekly from OpenAlex.

**Aesthetic direction:** Modernized Lancet editorial. Serif display + sans body, bright space, sharp red accent, card-led layout, premium agency polish.

**Motion direction:** Tasteful baseline with selected Expressive moments. GSAP + ScrollTrigger + SplitText.

## 2. Hard constraints (must not break)

1. **`publications.html` is fully regenerated by `generate_publications.py`** every Sunday via GitHub Actions. Restyling publications means editing the HTML strings inside the Python script.
2. **`<!-- BANNER_START -->` / `<!-- BANNER_END -->` markers** in `index.html` must remain — the generator regexes between them to refresh the ticker.
3. **CI workflow only commits `publications.html`** (file_pattern in `.github/workflows/update-publications.yml`). Local changes to `index.html` from the script are not pushed by CI.
4. **JS hooks that must keep working:** `filterPublications()`, `toggleAll()`, `#toTopBtn`, `.collapsible` toggle behavior — these run on the publications page.
5. **OpenAlex API** is the source of truth for citations/h-index/i10/publications list. Don't replace.
6. **Existing SEO must survive:** all meta tags, JSON-LD schema, `.sr-only` SEO article. Don't drop them.
7. **Static site only.** No build step, no bundler, no frameworks. Vanilla HTML/CSS/JS + GSAP via CDN.
8. **CSS variables used by the generator's inline styles:** `--golden-amber`, `--cactus-green`, `--dusty-blue`. These names must keep resolving to *something* sensible OR we remove the inline `style="..."` from the generator and let new classes do the work. **Decision: remove inline styles from generator, replace with new class names.**

**Backwards-compat aliases** (so generator inline styles don't break during transition):
```css
--golden-amber: var(--accent);
--cactus-green: var(--accent-2);
--dusty-blue: var(--ink-3);
```
These can be removed once the generator is updated.

### 3.2 Typography

| Family | Use | Weights |
|---|---|---|
| **Fraunces** (Google Fonts) | Display (h1, h2, hero name, metric numerals) | 500, 700 |
| **Inter** (Google Fonts) | Body, UI, paragraphs | 400, 500, 600, 700 |
| **JetBrains Mono** (Google Fonts) | Eyebrows, section numerals, micro-labels, code | 400, 500 |

**Scale (clamp-based, fluid):**
- h1: `clamp(36px, 6vw, 64px)` Fraunces 500, line-height 1.02, letter-spacing -1.5px
- h2: `clamp(24px, 3.5vw, 36px)` Fraunces 500, letter-spacing -.8px
- h3: 18-20px Fraunces 600
- body: 15-16px Inter 400, line-height 1.65
- eyebrow: 11px JetBrains Mono, uppercase, letter-spacing 2.5px, color `--accent`
- meta: 12px Inter 500, color `--ink-2`

### 3.3 Spacing & grid

- Container max-width: **1080px** (wider than current 1000px for more editorial breathing room)
- Gutter: 24px mobile, 36px desktop
- Section vertical rhythm: 80px top + bottom on desktop, 48px on mobile
- Card padding: 28px (was 40px — tighter, more grid-like)
- Border-radius: 4px on rectangles (portrait), 8px on cards, 999px on pill buttons. **No more 12-14px squishy radii.**

### 3.4 Components

- **Pill button** (rounded, 1.5px ink border, fills to ink on hover; primary variant is solid ink fills to accent)
- **Editorial card** (white bg, hairline border, hover lifts 3px + accent border)
- **Eyebrow label** ("// 01  INTRODUCTION" in mono, accent color)
- **Timeline row** (year on left in mono, content on right with hairline rule between)
- **Metric tile** (Fraunces numeral, mono label below, count-up on scroll)
- **Project card** (eyebrow tag + Fraunces title + body, hovers)
- **Ticker bar** (slim, full-bleed, ink background, paper text, marquee animation — replaces current babyblue marquee)

## 5. Motion choreography

Stack: **GSAP 3 + ScrollTrigger + SplitText** via CDN. Total payload ~50KB gzipped.

Honor `prefers-reduced-motion: reduce` — disable all GSAP timelines, keep only color transitions.

| Section | Animation | Trigger | Notes |
|---|---|---|---|
| Hero name | SplitText word-by-word reveal (stagger 70ms, y:24→0, opacity:0→1) | onLoad | The expressive moment of the hero |
| Hero portrait | Parallax translateY (-40px over scroll) | scroll | Tasteful, subtle |
| Hero scrubbed numeral | "2026" giant Fraunces in background, parallax + scrub | scroll | Background depth |
| Metrics | Count-up tween (0 → real value, 1.2s ease) | enter viewport (once) | The expressive moment of metrics |
| About / Now / Credentials / Expertise / Speaking / Affiliations cards | scrub fade-up (opacity 0→1, y 20→0) | scroll, scrubbed | Tasteful baseline |
| Sticky gutter labels | `position: sticky` per section, eyebrow stays pinned during section scroll | scroll | "// 01 INTRODUCTION" pattern |
| Research projects | **Pinned** section. Cards reveal sequentially as user scrolls past pinned container | pinned ScrollTrigger | The expressive moment of mid-page |
| Ticker | CSS marquee (keep existing keyframe approach, restyle slim/black) | always | Pause on hover |
| Buttons / links | Underline scaleX from right→left then left→right on hover | hover | Tasteful baseline |
| To-top button | Existing behavior, restyled | scroll | Keep |

## 7. Accessibility

- Color contrast: all body text vs `--paper` background ≥ 7:1; accent color is used for non-text decorative elements and small uppercase eyebrows (eyebrows are ≥4.5:1 against paper)
- Focus states: 2px `--accent` outline with 2px offset on all interactive elements
- `prefers-reduced-motion`: all GSAP timelines no-op; transitions reduced to 100ms color only
- Keyboard navigation: tab order matches visual order; collapsible publication years remain keyboard-toggleable
- Alt text on the portrait image
- Semantic structure preserved (one h1 per page, sections use `<section>`)

## 8. Performance budget

- Total page weight target: <300KB on initial load (excluding portrait image)
- GSAP + plugins: ~50KB gz via CDN
- Google Fonts: preconnect + subset (latin only)
- Defer GSAP and ScrollTrigger init until after first paint
- Lazy-load below-the-fold images if any are added
- No layout shift from font loading (use `font-display: swap` + size-adjust)

## 9. Out of scope (explicit YAGNI)

- Logo file/SVG generation for affiliations (text wordmarks only in v1)
- Dark mode toggle (not needed for v1)
- CMS / dynamic editing UI for the "Now" section (manual HTML edit is fine)
- Multi-language support
- Analytics integration
- Service worker / PWA
- Animations beyond the choreography table above (no Lottie, no WebGL)
- Restyling `receipts.html` / `receipts.js` (separate concern, not part of public portfolio)
- Auth-gated areas

## 10. Success criteria

- `index.html` renders all 12 sections in a real browser without console errors
- `python generate_publications.py` runs locally, regenerates `publications.html` with new styling, and updates the banner in `index.html` between the markers
- GitHub Actions weekly run continues to produce a clean `publications.html` commit
- All existing JS (search, expand/collapse, to-top) works on the publications page
- `prefers-reduced-motion: reduce` disables GSAP timelines
- Lighthouse desktop: ≥90 on Performance, Accessibility, Best Practices, SEO
- Page feels visibly more dynamic and editorial than the current version when scrolled through (subjective; verified by user review)
- Home-page metrics + selected publications stay fresh without needing a CI run (live OpenAlex fetch on page load, with static fallback if API fails)
